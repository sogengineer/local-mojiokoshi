# リアルタイム文字起こし & 議事録作成ワークフロー

## 入力情報

- 出力先: $1

## 実行手順

1. **出力先確認**: 出力先が指定されていない場合は、ユーザーに確認してください。
   - 出力先 ($1) が空の場合: 「議事録の保存先パスを入力してください（例: /path/to/meeting_notes.md）」

2. **リアルタイム文字起こし開始**: 以下のコマンドでリアルタイム文字起こしを開始
   ```bash
   uv run mojiokoshi realtime -o <一時テキストファイル.txt>
   ```

   ユーザーに以下を伝える：
   - 「リアルタイム文字起こしを開始します」
   - 「話し始めると文字起こしが表示されます」
   - 「終了するには Ctrl+C を押してください」

3. **文字起こし結果を読み込み**: Ctrl+Cで終了後、生成されたテキストファイルを読み込む

4. **誤字・誤認識の修正**: 文字起こし結果に対して以下の観点で修正を行う
   - **同音異義語**: 文脈から正しい漢字に修正（例：「以上」↔「異常」、「機会」↔「機械」）
   - **専門用語**: 業界用語やテクニカルタームを正しい表記に修正
   - **人名・固有名詞**: 会社名、製品名、人名などを文脈から推測して統一
   - **句読点**: 不自然な位置の句読点を修正し、読みやすく整形
   - **繰り返し・フィラー**: 「えー」「あのー」などの不要な言葉を削除
   - **文の区切り**: 適切な位置で文を区切り、段落を整理

5. **議事録サマリ作成**: 修正した文字起こし結果を以下の形式でMarkdownにまとめる
   ```markdown
   # 議事録

   ## 概要
   （会議の目的・主題を1-2文で）

   ## 参加者
   （判別できる場合のみ）

   ## 議論のポイント
   - ポイント1
   - ポイント2
   - ...

   ## 決定事項
   - 決定1
   - 決定2
   - ...

   ## アクションアイテム
   - [ ] タスク1（担当者）
   - [ ] タスク2（担当者）
   - ...

   ## 次回に向けて
   （次のステップや予定）

   ---

   ## 文字起こし全文（修正済み）
   （誤字修正後の文字起こしテキスト）
   ```

6. **ファイル保存**: 指定された出力先にMarkdownファイルを保存

7. **完了報告**: 保存したファイルパスをユーザーに報告
